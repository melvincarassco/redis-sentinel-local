Frequently Asked Questions

Got questions about local Sentinel execution? Here are some quick answers.

Why use a quorum threshold of 1 in local development?

In a standard production environment, you should run at least 3 Sentinels with a quorum of 2 to avoid split-brain issues. Locally, however, we use a quorum of 1 so that even if one sentinel is offline, the remaining instance can successfully trigger and execute the master failover cycle.

Why did my config files get rewritten automatically?

Redis Sentinel design requires updating the state to disk. Sentinel rewrites the sentinel.conf file to append discovered peer sentinels, list existing replicas, and save state variables. This is expected behavior.

How does my client driver connect to the Sentinels?

Rather than connecting directly to port 6379, your application client SDK (such as ioredis, go-redis, or Lettuce) is configured with the addresses of the Sentinel instances. The driver requests the address of billing-master from the active Sentinel, and then establishes connection to the master node automatically.
